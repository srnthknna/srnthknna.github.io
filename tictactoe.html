Minimax vs. Alpha-Beta Pruning

An unbeatable Tic-Tac-Toe opponent, built two ways: a textbook Minimax search, and the same search sped up with alpha–beta pruning. Both always find the same optimal move — the difference is how much of the game tree they have to look at to find it.

Two ways to search the same tree

Minimax

Recursively plays out every possible sequence of moves to the end of the game, then works backward: the computer's turns pick the highest score, the opponent's turns pick the lowest. Guaranteed optimal — but it looks at everything, including branches that could never affect the final choice.

Alpha-beta pruning

Same recursion, same guarantee — but it keeps a running "best score found so far" for each side. The moment a branch proves it can't beat what's already on the table, the search stops exploring it. Fewer states visited, identical decision.

The right branch's first outcome (2) is already worse for the computer than the left branch's guaranteed 3 — so whatever the second outcome is, it can only make the right branch look worse still. It's skipped entirely, and the computer plays left.
