tsapp: A Chained Time-Service Protocol

One Java program plays three roles — client, server, or proxy — for a simple time-sync protocol built on raw TCP and UDP sockets. A client can talk straight to the server, or through a chain of proxies. Every proxy the reply passes through stamps on the address it just heard back from and how long that took, so by the time it reaches the client, the message carries its own built-in traceroute.

The message format

Every request is a handful of fields packed into one string, delimited by a custom marker. What's included depends on what the client is asking for.

REQ0=UTC time
1=epoch time
2=set time
TYPt = TCP
u = UDP
USER / PASSonly for REQ 2
TIMEepoch value
to set

The reply works the same way in reverse: the server's answer gets one segment added per proxy hop on the way back, each holding a next-hop address and a round-trip time — read together, they reconstruct the path the message took.
