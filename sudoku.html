Sudoku Solver via Backtracking Search

Same generic search engine as the SAT solver next door — only the moves and the goal change. Here, a "move" is placing a digit in the next empty cell (restricted to values not already used in its row, column, or box), and "done" means the whole board is filled without conflicts.

The same ten lines

Nothing about the search itself is Sudoku-specific. Swap out what counts as a move and what counts as finished, and the exact same recursive engine that solves boolean expressions solves a 9×9 grid instead.

successors (SudokuConfig cells) = [ place digit in first empty cell | digit <- validCandidates ] isGoal config = every cell filled, no row/column/box conflicts
