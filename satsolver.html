A SAT Solver in a Few Lines of Search

Boolean satisfiability: given an expression built from AND, OR, and NOT, is there some assignment of true/false to its variables that makes the whole thing true? This one is built on a tiny generic backtracking engine — the same one, unmodified, also solves Sudoku on the next page.

One search engine, two problems

The generic engine only needs to know two things about a problem: what the possible next moves are (successors), and how to tell when you're done (isGoal). Everything else — the recursion, the backtracking, the "try the next option" logic — is written once.

solveAll isGoal c = let restSolutions = concat [solveAll isGoal c' | c' <- successors c] in if isGoal c then c:restSolutions else restSolutions

For SAT, a "move" is deciding the next variable's value; "done" means every variable is assigned and the expression evaluates to true.

Haskell implementation: a Config typeclass (successors + a goal test) shared across problems, with the boolean expression as an ordinary recursive BExp tree evaluated by a small interpreter.
